# Documentação do Projeto Task Manager Ágil

## Descrição do Projeto

O projeto Task Manager Ágil tem como objetivo simular o desenvolvimento de um sistema de gerenciamento de tarefas utilizando práticas de metodologias ágeis. O sistema permite o cadastro, consulta, atualização e exclusão de tarefas, representando um cenário comum em equipes que utilizam quadros Kanban para controle de atividades.

Este repositório foi estruturado para demonstrar conceitos de versionamento, organização de código, automação de testes e integração contínua, simulando um ambiente profissional de desenvolvimento de software.

## Metodologia Utilizada

A metodologia adotada neste projeto é inspirada no framework Scrum, combinada com práticas de Kanban para visualização do fluxo de trabalho. O desenvolvimento foi organizado em pequenas entregas incrementais, com versionamento contínuo e commits frequentes, simulando sprints curtos e evolução progressiva do sistema.

O quadro Kanban foi utilizado para organizar as atividades nas colunas To Do, In Progress e Done, permitindo o acompanhamento do progresso e a gestão visual das tarefas do projeto.

## Escopo do Sistema

O sistema contempla as seguintes funcionalidades principais:

- Cadastro de tarefas
- Listagem de tarefas
- Atualização de tarefas
- Remoção de tarefas
- Validação de regras básicas de negócio

Além disso, o projeto contempla a automação de testes e o uso de pipelines de integração contínua, garantindo qualidade e confiabilidade ao processo de desenvolvimento.

## Importância da Modelagem
A modelagem do sistema é essencial para a compreensão das regras de negócio e das relações entre as entidades envolvidas. Através dos diagramas UML, é possível visualizar os atores, casos de uso e a estrutura das classes do sistema, facilitando o entendimento, manutenção e evolução do software.
