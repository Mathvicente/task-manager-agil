## Mudança de Escopo

Durante o desenvolvimento do projeto, foi identificada a necessidade de melhorar a priorização das atividades, simulando um cenário real de adaptação de requisitos em projetos ágeis.

Inicialmente, o sistema permitia apenas o cadastro de tarefas com título, descrição e status. Como melhoria, foi adicionada a funcionalidade de **prioridade de tarefas**, permitindo classificar cada tarefa como **Alta, Média ou Baixa**.

Essa mudança impactou:
- O serviço de tarefas (src/task_service.py)
- Os testes automatizados (tests/test_task_service.py)
- A documentação UML (docs)

A funcionalidade foi implementada de forma incremental, seguindo o conceito de mudança de escopo comum em projetos ágeis.
